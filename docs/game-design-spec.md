# 犬タローの虫さんまってまって 仕様・設計資料

最終更新: 2026-06-19

## 1. 概要

「犬タローの虫さんまってまって」は、犬タローを操作して、制限時間内に虫さんをできるだけ多く捕まえるブラウザゲームです。

プレイヤーは左右移動とジャンプを使い、左から流れてくる虫さんを集めます。空には鳥の敵キャラクターが飛んでおり、不定期にお邪魔アイテムを落としてきます。お邪魔に一度でも当たると即ゲームオーバーです。

公開URL:

https://erinui.github.io/inutaro-game/

ゲームURL:

https://erinui.github.io/inutaro-game/games/inutaro-mushi/

最新確認URL:

http://127.0.0.1:8778/index.html?home-local=20260619

## サイト構成

GitHub Pagesのルートは、ゲーム単体ページではなく、犬タローゲームのホームとして使用します。

ホームにはゲームカード、SNS導線、キャラクター紹介、ブログ、利用規約、プライバシーポリシーへの導線を配置します。今後ゲームが増えた場合は、ゲームカードを追加して選択できる構成にします。

ホームのおしらせ欄は、`data/news.json` を読み込んで新着カードを表示します。X、YouTube、ブログ、ゲーム更新を同じカードUIで扱えるようにしておき、将来的にGitHub ActionsなどでSNS/YouTubeの新着取得を自動化しても、表示側の実装を大きく変えずに済む構成にします。

ホームのビジュアルは、犬タローの手描き感に合わせて、明るい背景、ステッカー風のラベル、色分けされたカード、立体感のあるボタンでポップに見せます。ゲーム本体は操作しやすさを優先したまま、ホーム側で遊びに行きたくなる雰囲気を作ります。

## 2. 企画コンセプト

### ゲーム名

犬タローの虫さんまってまって

### キャラクター

犬タローは、このゲームでプレイヤーが操作するキャラクターです。

犬タローは語尾に「ヌ」が付くキャラクター設定です。お邪魔命中時の短いセリフも、この設定に合わせて `ヌッ！` と表示します。

### 体験の方向性

- やさしい手描き調の世界観
- 虫さんを追いかけて集める、軽く遊べるアクション
- 鳥が落とすお邪魔に注意する緊張感
- 失敗してもすぐ再挑戦できる短時間プレイ
- スマートフォンでも遊びやすいスティック操作とジャンプボタン

### 想定プレイ感

基本操作はシンプルにし、プレイヤーが「虫さんを追いかける」ことに集中できる設計です。蝶はふわふわ、カタツムリはゆっくり、トンボは素早く緩急のある動きにすることで、同じ「取る」行為の中に違うリズムを作っています。

### SP操作

スマートフォンでは、画面下部にスティックとジャンプボタンを表示します。

- 右側: 左右移動スティック
- 左側: ジャンプボタン
- ゲーム中の右上はサウンドボタンを優先し、リセットボタンは表示しない
- スティックとジャンプボタンは中心の高さを揃える
- スティック操作領域は操作性向上のため初期SP実装より10%拡大
- ゲーム領域ではダブルタップ拡大、長押しメニュー、テキスト選択、ドラッグ開始を抑制

横移動速度は、操作性改善のため初期実装よりも抑えています。キーボード移動、SPスティック移動、目標位置への追従速度のすべてを現在の調整値で統一しています。

### 初期画面の挙動

ゲーム開始前でも、画面クリックで犬タローがジャンプします。これは、開始前にキャラクターの反応を見せるための演出です。

初期画面では `はじめる` ボタンは画面中央に配置します。ゲーム中は右上にサウンドボタンを置き、リセットボタンは表示しません。

### 表記方針

プレイヤーに見える主要UIは日本語表記に統一します。ゲームの状態表示は `クリア！`、`ゲームオーバー`、ボタンは `はじめる`、`けっかへ`、`もう一回`、`タイトルへ` を使用します。サウンドボタンはOS依存の絵文字ではなく、CSSで描画した音符アイコンを使用します。

## 6. 虫さんアイテム仕様

| 種類 | 画像 | 動き | 速度 | 出現位置 |
| --- | --- | --- | --- | --- |
| カタツムリ | `assets/item_a.png` | 地面をゆっくり進む | 遅い | 地面付近 |
| 蝶 | `assets/item_b.png` | 空中を上下にふわふわ進む | 中程度 | ジャンプで届く範囲を基準に調整 |
| トンボ | `assets/item_c.png` | 水平移動、急停止、ホバリング、急発進 | 速い | ギリギリ届く高さを上限に調整 |

### 到達可能性の設計

画面サイズや縦横比が変わると、ジャンプで届く高さも見た目上変わります。そのため、空中アイテムの高さは固定値ではなく、現在のキャンバスサイズとジャンプ上限から計算します。

蝶は上下に動くため、必ず「最低位置」がジャンプ上限より低い位置になるようにしています。つまり、動きの途中で届きにくい瞬間があっても、必ず捕まえられる高さまで降りてきます。

トンボは上下移動しないため、発生位置そのものをジャンプでギリギリ届く高さ以下に制限しています。現在は `reachableAirRange()` と `maxPlayerLift()` で実際のジャンプ物理から到達可能範囲を計算し、その範囲内で発生させます。

## 8. 演出仕様

### 捕獲時エフェクト

アイテムを捕まえた時は、黄色いキラキラを軽量表示します。

- 黄色い十字2つ
- 黄色い菱形1つ
- 合計3個のみ
- 発光、ぼかし、加算合成は使用しない

過去の強い発光演出はパフォーマンス影響が大きかったため、現在は以前の楕円粒子に近い軽さで、形だけキラキラに変更した演出にしています。

### 着地・通常粒子

ジャンプや着地時は、従来の小さな粒子演出を使用します。

### 効果音

| タイミング | ファイル |
| --- | --- |
| ジャンプ時 | `assets/sound_jump.mp3` |
| アイテム取得時 | `assets/sound_itemget.mp3` |
| お邪魔命中時 | `assets/sound_hazard-hit.mp3` |

ブラウザの自動再生制限に配慮し、初回のポインター操作またはキーボード操作時に音声を読み込むようにしています。再生に失敗してもゲーム進行は止めません。

### BGM

| タイミング | ファイル |
| --- | --- |
| ロード完了後からページ滞在中 | `assets/bgm.mp3` |

BGMは長尺音源のため、効果音用のWeb Audio APIではなく `HTMLAudioElement` のループ再生を使用します。ゲーム画像のロード完了後、低優先タイミングで再生開始を試み、クリア / ゲームオーバー / リザルト画面でも止めずに流し続けます。ブラウザの自動再生制限で止められた場合は、最初のポインター操作またはキーボード操作時に再開します。効果音を邪魔しないように音量は控えめに設定します。

画面右上のサウンドボタンで、BGMと効果音を一括でオン/オフできます。オフ時はBGMを一時停止し、効果音は再生処理を行いません。オンに戻した場合はBGMを続きから再開します。ゲーム中の右上リセットボタンは表示せず、サウンドボタンを優先して配置します。表示は絵文字ではなくCSSの音符アイコンで描画し、端末ごとの見た目の差を抑えます。

### クリア / ゲームオーバー

ゲーム終了時は一度ゲーム画面上で状態を認識させ、`けっかへ` ボタンからリザルトモーダルへ進みます。ゲームオーバー時は短い停止時間を入れ、着弾画像を残すことで失敗理由が分かりやすくなるようにしています。

### ゲームループ

`requestAnimationFrame` を使い、毎フレーム以下を実行します。

1. 経過時間を計算
2. プレイヤー状態を更新

5. アイテムとお邪魔の位置更新
6. 衝突判定
7. 背景、敵、アイテム、お邪魔、犬タロー、演出を描画

### 衝突判定

アイテム取得は、犬タローのキャッチ範囲内にアイテム中心が入ったかで判定します。

お邪魔命中は、犬タローの体判定とお邪魔の矩形判定が重なったかで判定します。

### 画像読み込み

画像は `new Image()` で読み込み、読み込み完了後にCanvasへ描画します。読み込み中に草案段階の図形が一瞬見える問題を避けるため、画像群の読み込みを待ってからゲームループを開始します。

読み込みが間に合わない、または失敗した場合でも、最低限の図形描画でフォールバックできる構造があります。

### SPパフォーマンス

iPhone Safariでは高DPR環境でCanvas内部解像度が過剰に大きくなりやすいため、Canvasの描画倍率に上限を設けています。PCでは最大DPR 2、SPなど粗いポインター環境では最大DPR 1.5に制限し、描画ピクセル数を抑えます。背景画像は画面サイズに合わせたキャッシュCanvasへ一度描画し、ゲームループ中はキャッシュを再利用します。残り時間表示などのDOM更新は、値が変化した時だけ行います。

描画ループは状態に応じて間引きます。プレイ中は60fps相当を維持し、タイトル画面や `けっかへ` ボタン待ちは30fps相当、リザルトモーダル表示中は15fps相当に落とします。60Hz端末でフレーム境界の誤差により誤って間引かないよう、プレイ中のフレーム判定には小さな許容幅を持たせます。タブが非表示になった場合は更新・描画を停止し、復帰時に経過時間を補正してゲーム速度やジャンプ挙動が跳ねないようにします。

効果音は `HTMLAudioElement` の都度リセット再生ではなく、Web Audio APIで初回操作時に読み込み・デコードしてから再生します。アイテム取得音は約2.9秒と長いため、ゲーム中では短い効果音として再生時間を制限し、連続取得時に音が重なり続けないようにしています。SafariでWeb Audioの解除やデコードに失敗した場合は、HTMLAudioへフォールバックします。フォールバックAudioはアクセス時に先読みせず、解除目的で実際に再生もしないことで、アクセス時や初回操作時に効果音がまとめて鳴る事故を避けます。

BGMは約53秒の音源を128kbps / 44.1kHzへ軽量化し、長時間ループ再生しても効果音処理やCanvas描画に干渉しにくいよう、効果音とは別にHTMLAudioのループ再生で扱います。初期表示中のパフォーマンスを優先するため、BGMの読み込み開始は画像ロード完了後まで遅らせます。

### 保存・共有

CanvasからPNGを生成し、以下の用途で利用します。

- クリア / ゲームオーバー画面キャプチャ
- リザルト共有用カード画像
- 画像付き共有

リザルト保存は、画面上のリザルトモーダルそのものではなく、共有用に整えた1200x630pxの専用カード画像をCanvasで生成します。ボタン文言は `けっかを保存` のままですが、保存・共有されるリザルト画像は以前の共有用フォーマットです。

共有には `navigator.share()` と `File` を使用します。対応環境では画像ファイルを共有シートへ渡し、非対応環境ではリンクコピーやダウンロードへフォールバックします。

現在の共有文は以下です。

| 終了理由 | 共有文 |
| --- | --- |
| クリア | `犬タローの虫さんまってまってで虫さんを{合計}ひき捕まえたよ！` |
| ゲームオーバー | `犬タローの虫さんまってまってで虫さんを{合計}ひき捕まえたよ！{秒数}秒でゲームオーバー…もう一回！` |

LINEやSNSのURLプレビューで正式なサイト名・ゲーム名・説明文が表示されるように、ホームとゲームページそれぞれにOGP/Twitter Cardメタ情報を設定します。共有先のプレビューキャッシュが古い情報を保持する場合があるため、ゲーム内共有URLには更新用クエリを付けています。

### キャッシュ対策

GitHub Pagesやブラウザのキャッシュで古いJavaScript/CSSが残らないように、HTML側で以下のようにバージョン付きURLを使用しています。

- `home.css?v=...`
- `home.js?v=...`
- `style.css?v=...`
- `game.js?v=...`
- `assets/bg.jpg?v=...`

現在のローカル確認版:

- `home.css?v=20260619-home-local`
- `home.js?v=20260619-news-local`
- `style.css?v=20260617-note-icon`
- `game.js?v=20260619-home-local`
