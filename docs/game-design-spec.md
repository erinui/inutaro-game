# 犬タローの虫さんまってまって 仕様・設計資料

最終更新: 2026-06-16

## 1. 概要

「犬タローの虫さんまってまって」は、犬タローを操作して、制限時間内に虫さんをできるだけ多く捕まえるブラウザゲームです。

プレイヤーは左右移動とジャンプを使い、左から流れてくる虫さんを集めます。空には鳥の敵キャラクターが飛んでおり、不定期にお邪魔アイテムを落としてきます。お邪魔に一度でも当たると即ゲームオーバーです。

公開URL:

https://erinui.github.io/inutaro-game/

最新確認URL:

https://erinui.github.io/inutaro-game/?stick=20260615b

## 2. 企画コンセプト

### ゲーム名

犬タローの虫さんまってまって

### キャラクター

犬タローは、このゲームでプレイヤーが操作するキャラクターです。

### 体験の方向性

- やさしい手描き調の世界観
- 虫さんを追いかけて集める、軽く遊べるアクション
- 鳥が落とすお邪魔に注意する緊張感
- 失敗してもすぐ再挑戦できる短時間プレイ
- スマートフォンでも遊びやすいスティック操作とジャンプボタン

### 想定プレイ感

基本操作はシンプルにし、プレイヤーが「虫さんを追いかける」ことに集中できる設計です。蝶はふわふわ、カタツムリはゆっくり、トンボは素早く緩急のある動きにすることで、同じ「取る」行為の中に違うリズムを作っています。

## 3. ゲームルール

### 目的

制限時間内に虫さんをできるだけ多く捕まえること。

### 終了条件

| 条件 | 結果 |
| --- | --- |
| 制限時間を最後まで遊びきる | CLEAR |
| お邪魔アイテムに1回当たる | GAME OVER |

### 制限時間

40秒

### スコア

捕まえた虫さん1ひきにつき、TOTALが1増えます。

### リザルト表示

ゲーム終了後、すぐにリザルトを出さず、まず画面上に `CLEAR` または `GAME OVER` を表示します。その後、`RESULT` ボタンを押すとリザルトモーダルが表示されます。

| 表示項目 | CLEAR時 | GAME OVER時 |
| --- | --- | --- |
| 結果タイトル | CLEAR | GAME OVER |
| TIME | 非表示 | 何秒でゲームオーバーになったか表示 |
| TOTAL | 総獲得数を「ひき」単位で表示 | 総獲得数を「ひき」単位で表示 |
| アイテム別獲得数 | 各虫画像と「ひき」単位で表示 | 各虫画像と「ひき」単位で表示 |

### リザルト操作

| ボタン | 内容 |
| --- | --- |
| 画面保存 | CLEAR / GAME OVER画面のキャプチャを保存 |
| リザルト保存 | リザルトカード画像を保存 |
| 共有 | 共有パネルを開く |
| 画像つき共有 | 画面キャプチャ、リザルト画像、ゲームURLを端末の共有機能へ渡す |
| URLコピー | 共有文とゲームURLをクリップボードへコピー |
| フォロー | 犬タロー関連のX / YouTubeリンクを表示 |
| RESTART | ゲームを最初からやり直す |
| おわり | タイトル状態へ戻る |

SP時の画像保存は、ブラウザの制限により写真アプリへ直接無音保存はできません。対応端末ではネイティブ共有/保存シートを開き、ユーザーが写真アプリ保存を選べる導線にしています。PCでは通常のPNGダウンロードになります。

## 4. 操作仕様

| 操作 | 内容 |
| --- | --- |
| STARTボタン | ゲーム開始 |
| RESET / RESTARTボタン | ゲームを最初からやり直す |
| おわりボタン | タイトル状態へ戻る |
| RESULTボタン | 終了後にリザルトを表示 |
| マウス / タッチ移動 | PC時は犬タローの左右移動位置を指定 |
| 画面クリック / タップ | PC時はジャンプ |
| キーボード左 / A | 左へ移動 |
| キーボード右 / D | 右へ移動 |
| Space / 上 / W | ジャンプ |
| SPスティック | 左右移動 |
| SPジャンプボタン | ジャンプ |

### SP操作

スマートフォンでは、画面下部にスティックとジャンプボタンを表示します。

- 右側: 左右移動スティック
- 左側: ジャンプボタン
- RESETボタンはPCと同様に右上へ小さめに配置
- スティックとジャンプボタンは中心の高さを揃える
- スティック操作領域は操作性向上のため初期SP実装より10%拡大
- ゲーム領域ではダブルタップ拡大、長押しメニュー、テキスト選択、ドラッグ開始を抑制

横移動速度は、操作性改善のため初期実装よりも抑えています。キーボード移動、SPスティック移動、目標位置への追従速度のすべてを現在の調整値で統一しています。

### 初期画面の挙動

ゲーム開始前でも、画面クリックで犬タローがジャンプします。これは、開始前にキャラクターの反応を見せるための演出です。

初期画面では `START` ボタンは画面中央に配置します。ゲーム開始後は、操作や視界の邪魔にならない右上へ移動します。

## 6. 虫さんアイテム仕様

| 種類 | 画像 | 動き | 速度 | 出現位置 |
| --- | --- | --- | --- | --- |
| カタツムリ | `assets/item_a.png` | 地面をゆっくり進む | 遅い | 地面付近 |
| 蝶 | `assets/item_b.png` | 空中を上下にふわふわ進む | 中程度 | ジャンプで届く範囲を基準に調整 |
| トンボ | `assets/item_c.png` | 水平移動、急停止、ホバリング、急発進 | 速い | ギリギリ届く高さを上限に調整 |

### 到達可能性の設計

画面サイズや縦横比が変わると、ジャンプで届く高さも見た目上変わります。そのため、空中アイテムの高さは固定値ではなく、現在のキャンバスサイズとジャンプ上限から計算します。

蝶は上下に動くため、必ず「最低位置」がジャンプ上限より低い位置になるようにしています。つまり、動きの途中で届きにくい瞬間があっても、必ず捕まえられる高さまで降りてきます。

トンボは上下移動しないため、発生位置そのものをジャンプでギリギリ届く高さ以下に制限しています。現在は `reachableAirRange()` と `maxPlayerLift()` で実際のジャンプ物理から到達可能範囲を計算し、その範囲内で発生させます。

## 8. 演出仕様

### 捕獲時エフェクト

アイテムを捕まえた時は、黄色いキラキラを軽量表示します。

- 黄色い十字2つ
- 黄色い菱形1つ
- 合計3個のみ
- 発光、ぼかし、加算合成は使用しない

過去の強い発光演出はパフォーマンス影響が大きかったため、現在は以前の楕円粒子に近い軽さで、形だけキラキラに変更した演出にしています。

### 着地・通常粒子

ジャンプや着地時は、従来の小さな粒子演出を使用します。

### 効果音

| タイミング | ファイル |
| --- | --- |
| ジャンプ時 | `assets/sound_jump.mp3` |
| アイテム取得時 | `assets/sound_itemget.mp3` |
| お邪魔命中時 | `assets/sound_hazard-hit.mp3` |

ブラウザの自動再生制限に配慮し、初回のポインター操作またはキーボード操作時に音声を読み込むようにしています。再生に失敗してもゲーム進行は止めません。

### CLEAR / GAME OVER

ゲーム終了時は一度ゲーム画面上で状態を認識させ、`RESULT` ボタンからリザルトモーダルへ進みます。GAME OVER時は短い停止時間を入れ、着弾画像を残すことで失敗理由が分かりやすくなるようにしています。

## 9. 画面仕様

### 画面構成

- Canvasによるゲーム画面
- 左上HUD
  - ITEMS
  - TIME
- 初期画面中央STARTボタン
- ゲーム中のRESETボタン
- 終了後のRESULTボタン
- リザルトモーダル
- SP時のスティック操作UIとジャンプボタン

### 背景

背景画像は `assets/bg.jpg` を使用します。

Canvas全面に対して、画像をcover相当で描画します。画面比率が変わっても隙間が出ないように、画像の中央を基準にトリミング表示します。

### レスポンシブ対応

画面幅720px以下では、縦長画面に合わせてゲーム領域を `10 / 16` に変更します。HUDやボタン、リザルト表示も狭い画面に収まるサイズへ調整します。

リザルトモーダルは内容が増えても操作できるように、縦方向スクロールに対応しています。

## 10. 技術設計

### 構成

| ファイル | 役割 |
| --- | --- |
| `index.html` | HTML構造、HUD、ボタン、SP操作UI、リザルトモーダル |
| `style.css` | レイアウト、ボタン、HUD、リザルト、SP操作UIの見た目 |
| `game.js` | ゲームループ、入力、物理、出現、衝突、描画、保存/共有 |
| `assets/` | キャラクター、アイテム、背景画像 |
| `docs/game-design-spec.md` | 仕様・設計・開発記録 |

### 使用技術

- HTML
- CSS
- JavaScript
- Canvas 2D API
- GitHub Pages

外部ライブラリは使用していません。静的ファイルのみで動作するため、GitHub Pagesにそのまま配置できます。

### ゲームループ

`requestAnimationFrame` を使い、毎フレーム以下を実行します。

1. 経過時間を計算
2. プレイヤー状態を更新
3. 敵の移動を更新
4. アイテムとお邪魔の出現
5. アイテムとお邪魔の位置更新
6. 衝突判定
7. 背景、敵、アイテム、お邪魔、犬タロー、演出を描画

### 状態管理

ゲーム状態は `state` オブジェクトで管理します。

主な状態:

- 実行中かどうか
- 開始時刻
- 終了理由
- 残り時間
- スコア
- アイテム別獲得数
- 犬タローの位置とジャンプ状態
- 敵の位置
- アイテム出現バッグ
- 画面上のアイテム一覧
- お邪魔一覧
- 着弾演出一覧
- 捕獲/着地粒子一覧
- SPスティック入力状態

### 衝突判定

アイテム取得は、犬タローのキャッチ範囲内にアイテム中心が入ったかで判定します。

お邪魔命中は、犬タローの体判定とお邪魔の矩形判定が重なったかで判定します。

### 画像読み込み

画像は `new Image()` で読み込み、読み込み完了後にCanvasへ描画します。読み込み中に草案段階の図形が一瞬見える問題を避けるため、画像群の読み込みを待ってからゲームループを開始します。

読み込みが間に合わない、または失敗した場合でも、最低限の図形描画でフォールバックできる構造があります。

### 保存・共有

CanvasからPNGを生成し、以下の用途で利用します。

- CLEAR / GAME OVER画面キャプチャ
- リザルトカード
- 画像付き共有

共有には `navigator.share()` と `File` を使用します。対応環境では画像ファイルを共有シートへ渡し、非対応環境ではURLコピーやダウンロードへフォールバックします。

### キャッシュ対策

GitHub Pagesやブラウザのキャッシュで古いJavaScript/CSSが残らないように、`index.html` 側で以下のようにバージョン付きURLを使用しています。

- `style.css?v=...`
- `game.js?v=...`
- `assets/bg.jpg?v=...`

現在の公開確認版:

- `style.css?v=20260616-touch-guard`
- `game.js?v=20260616-touch-guard`
