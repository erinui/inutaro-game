# 犬タローの虫さんまってまって 仕様・設計資料

## 1. 概要

「犬タローの虫さんまってまって」は、犬タローを操作して、制限時間内に虫さんをできるだけ多く捕まえるブラウザゲームです。

プレイヤーは左右移動とジャンプを使い、左から流れてくる虫さんを集めます。空には鳥の敵キャラクターが飛んでおり、不定期にお邪魔アイテムを落としてきます。お邪魔に一度でも当たると即ゲームオーバーです。

公開URL:

https://erinui.github.io/inutaro-game/

## 2. 企画コンセプト

### ゲーム名

犬タローの虫さんまってまって

### キャラクター

犬タローは、このゲームでプレイヤーが操作するキャラクターです。

### 体験の方向性

- やさしい手描き調の世界観
- 虫さんを追いかけて集める、軽く遊べるアクション
- 鳥が落とすお邪魔に注意する緊張感
- 失敗してもすぐ再挑戦できる短時間プレイ

### 想定プレイ感

基本操作はシンプルにし、プレイヤーが「虫さんを追いかける」ことに集中できる設計です。蝶はふわふわ、カタツムリはゆっくり、トンボは素早く緩急のある動きにすることで、同じ「取る」行為の中に違うリズムを作っています。

## 3. ゲームルール

### 目的

制限時間内に虫さんをできるだけ多く捕まえること。

### 終了条件

| 条件 | 結果 |
| --- | --- |
| 制限時間を最後まで遊びきる | CLEAR |
| お邪魔アイテムに1回当たる | GAME OVER |

### 制限時間

40秒

### スコア

捕まえた虫さん1ひきにつき、TOTALが1増えます。

### リザルト表示

ゲーム終了後、すぐにリザルトを出さず、まず画面上に `CLEAR` または `GAME OVER` を表示します。その後、`RESULT` ボタンを押すとリザルトモーダルが表示されます。

| 表示項目 | CLEAR時 | GAME OVER時 |
| --- | --- | --- |
| 結果タイトル | CLEAR | GAME OVER |
| TIME | 非表示 | 何秒でゲームオーバーになったか表示 |
| TOTAL | 総獲得数を「ひき」単位で表示 | 総獲得数を「ひき」単位で表示 |
| アイテム別獲得数 | 各虫画像と「ひき」単位で表示 | 各虫画像と「ひき」単位で表示 |

## 4. 操作仕様

| 操作 | 内容 |
| --- | --- |
| STARTボタン | ゲーム開始 |
| RESET / RESTARTボタン | ゲームを最初からやり直す |
| RESULTボタン | 終了後にリザルトを表示 |
| マウス / タッチ移動 | 犬タローの左右移動位置を指定 |
| 画面クリック / タップ | ジャンプ |
| キーボード左 / A | 左へ移動 |
| キーボード右 / D | 右へ移動 |
| Space / 上 / W | ジャンプ |

### 初期画面の挙動

ゲーム開始前でも、画面クリックで犬タローがジャンプします。これは、開始前にキャラクターの反応を見せるための演出です。

初期画面では `START` ボタンは画面中央に配置します。ゲーム開始後は、操作や視界の邪魔にならない位置へ移動します。

## 6. 虫さんアイテム仕様

| 種類 | 画像 | 動き | 速度 | 出現位置 |
| --- | --- | --- | --- | --- |
| カタツムリ | `assets/item_a.png` | 地面をゆっくり進む | 遅い | 地面付近 |
| 蝶 | `assets/item_b.png` | 空中を上下にふわふわ進む | 中程度 | ジャンプで届く範囲を基準に調整 |
| トンボ | `assets/item_c.png` | 水平移動、急停止、ホバリング、急発進 | 速い | ジャンプ頂点と同じか、それより下 |

### 到達可能性の設計

画面サイズや縦横比が変わると、ジャンプで届く高さも見た目上変わります。そのため、空中アイテムの高さは固定値ではなく、現在のキャンバスサイズとジャンプ上限から計算します。

蝶は上下に動くため、必ず「最低位置」がジャンプ上限より低い位置になるようにしています。つまり、動きの途中で届きにくい瞬間があっても、必ず捕まえられる高さまで降りてきます。

トンボは上下移動しないため、発生位置そのものをジャンプ頂点と同じか、それより下に制限しています。

## 8. 画面仕様

### 画面構成

- Canvasによるゲーム画面
- 左上HUD
  - ITEMS
  - TIME
- 初期画面中央STARTボタン
- ゲーム中のRESETボタン
- 終了後のRESULTボタン
- リザルトモーダル

### 背景

背景画像は `assets/bg.jpg` を使用します。

Canvas全面に対して、画像をcover相当で描画します。画面比率が変わっても隙間が出ないように、画像の中央を基準にトリミング表示します。

### レスポンシブ対応

画面幅720px以下では、縦長画面に合わせてゲーム領域を `10 / 16` に変更します。HUDやボタン、リザルト表示も狭い画面に収まるサイズへ調整します。

## 9. 技術設計

### 構成

| ファイル | 役割 |
| --- | --- |
| `index.html` | HTML構造、HUD、ボタン、リザルトモーダル |
| `style.css` | レイアウト、ボタン、HUD、リザルトの見た目 |
| `game.js` | ゲームループ、入力、物理、出現、衝突、描画 |
| `assets/` | キャラクター、アイテム、背景画像 |

### 使用技術

- HTML
- CSS
- JavaScript
- Canvas 2D API
- GitHub Pages

外部ライブラリは使用していません。静的ファイルのみで動作するため、GitHub Pagesにそのまま配置できます。

### ゲームループ

`requestAnimationFrame` を使い、毎フレーム以下を実行します。

1. 経過時間を計算
2. プレイヤー状態を更新
3. 敵の移動を更新
4. アイテムとお邪魔の出現
5. アイテムとお邪魔の位置更新
6. 衝突判定
7. 背景、敵、アイテム、お邪魔、犬タロー、演出を描画

### 状態管理

ゲーム状態は `state` オブジェクトで管理します。

主な状態:

- 実行中かどうか
- 開始時刻
- 終了理由
- 残り時間
- スコア
- アイテム別獲得数
- 犬タローの位置とジャンプ状態
- 敵の位置
- 画面上のアイテム一覧
- お邪魔一覧
- 着弾演出一覧

### 衝突判定

アイテム取得は、犬タローのキャッチ範囲内にアイテム中心が入ったかで判定します。

お邪魔命中は、犬タローの体判定とお邪魔の矩形判定が重なったかで判定します。

### 画像読み込み

画像は `new Image()` で読み込み、読み込み完了後はCanvasに描画します。読み込みが間に合わない場合でも、最低限の図形描画でフォールバックできる構造があります。

### キャッシュ対策

GitHub Pagesやブラウザのキャッシュで古いJavaScript/CSSが残らないように、`index.html` 側で以下のようにバージョン付きURLを使用しています。

- `style.css?v=...`
- `game.js?v=...`
- `assets/bg.jpg?v=...`
