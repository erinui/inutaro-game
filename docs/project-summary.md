## 4. 機能まとめ

### 親サイト機能

| 機能 | 内容 |
| --- | --- |
| 親ホーム | 現在用意できているページへの入口 |
| ゲーム一覧 | 公開中・準備中ゲームをカードで表示 |
| 外部リンク | X / YouTubeを別タブで開く |
| OGP | 親ホーム、ゲーム一覧、ゲーム本体でメタ情報を設定 |
| アイコン | favicon、apple-touch-icon、192pxアイコンを設定 |

### ゲーム機能

| 機能 | 内容 |
| --- | --- |
| 基本ルール | 40秒以内に虫さんをできるだけ捕まえる |
| 操作 | PCはマウス/クリック/キーボード、SP横向きはスティック+ジャンプボタン |
| アイテム | カタツムリ、蝶、トンボ |
| 危険要素 | カラスの落とし物に一度当たるとゲームオーバー |
| 終了表示 | クリア / ゲームオーバー後、`けっかへ` でリザルトへ進む |
| リザルト | 合計、虫さん別の獲得数、「ひき」単位表示 |
| 保存 | 画面保存、リザルト共有用画像保存 |
| 共有 | 画像付き共有、リンクコピー |
| フォロー | X / YouTubeリンク表示 |
| サウンド | BGM、ジャンプ、虫さん取得、ダメージ効果音 |
| サウンド切替 | 音符ボタンでBGM・効果音を一括ON/OFF |

### ゲームバランス・操作改善

これまでに以下を調整しています。

- 犬タローの左右移動速度を抑制
- SP横向き用スティックとジャンプボタンを追加
- 固定横画面モードを採用し、端末の画面回転設定に依存しない表示へ変更
- 固定横画面モードでは、縦向きビューポート時のCanvas描画サイズとタッチ座標を回転表示に合わせて変換
- SPブラウザUI縮小補助として、ゲーム画面を固定したまま見えないスクロール余白と開始時の軽いscrollToを追加
- SPでダブルタップ拡大や長押し選択が起きないよう抑制
- トンボの出現高さをジャンプで届く範囲に調整
- 蝶の最低高度をジャンプで届く範囲に調整
- 虫さん出現の偏りを抑えるため、バッグ方式で平均化
- カラスの落とし物のサイズ、当たり判定、落下速度を調整
- 捕獲エフェクトを軽量なキラキラ表現に変更

## 5. デザイン方針

### サイト側

- 明るい背景
- 手描き感のある世界観
- 角丸8px前後のカード
- ステッカー風のラベル
- 立体感のあるボタン
- 表示フォントはサイト全体で `けいなんポップ体` に統一
- PC/SPどちらでもリンクが押しやすい構成

### ゲーム側

- 画面を見れば遊び方がわかるシンプルさ
- スタート画面に「あそびかた」ボタン
- あそびかたはPC/SP横向きで操作説明を出し分け
- リザルトはスクロール対応
- ゲーム中は情報量を抑え、操作に集中しやすくする

### Figma

Figmaファイル「犬タロー」に、現在のホームデザインを以下のフレームとして作成済みです。

| フレーム | 内容 |
| --- | --- |
| `Home / PC 1440 - current design` | PC版ホームデザイン |
| `Home / SP 390 - current design` | SP版ホームデザイン |

Figma MCPのStarterプランの呼び出し上限により、最後の細部調整は一部未完了です。ただし、デザイン方向性の確認用としては使用できます。

現在は `erinui/inutaro-game` リポジトリで開発・公開しています。

| ブランチ | 役割 |
| --- | --- |
| `main` | ソース管理 |

### 反映時の基本工程

1. ローカルで実装
2. ローカルURLでPC/SP表示確認
3. 必要に応じて実機確認
4. 変更対象のみステージング
5. `main` にコミット・プッシュ
6. 公開対象ファイルを `gh-pages` に反映
7. GitHub Pages公開URLで確認

## 9. 開発工程の流れ

これまでの大きな流れです。

1. 参考ゲームの確認
2. 犬タロー操作ゲームとして作り直し
3. プレイヤー、カラス、虫さん、背景素材の反映
4. ゲームルールを確定
5. 当たり判定、出現位置、移動速度を調整
6. リザルト、保存、共有、フォロー導線を追加
7. SP横向き操作UIを追加
8. 音声、BGM、サウンド切替を追加
9. パフォーマンス改善
10. あそびかたモーダルを追加
11. ホームページを追加
12. Figmaにホームデザインを作成
13. `erinui` 親ホーム、`/games/` ゲーム一覧、ゲーム本体の階層へ整理

## 10. 今後の移行方針

最終的に `https://erinui.github.io/` を親サイトにする場合は、以下の順で進めます。

1. `erinui/erinui.github.io` リポジトリを用意
2. 現在の親ホーム、ゲーム一覧、固定ページ、ゲーム本体を移植
3. OGP URLと共有URLを `https://erinui.github.io/` 前提に統一
4. `erinui/inutaro-game` 側には新URL案内を残す
5. 実機で親ホーム、ゲーム一覧、ゲーム本体、共有、保存を確認
6. 必要に応じて独自ドメインやアクセス解析を検討
